# Task Card 005: Repo Structure Audit and Target Layout

## Goal
Audit the current repository structure and define a cleaner target layout for ongoing development.

---

## Scope
- inspect current backend folders and module placement
- identify confusing or mixed-responsibility areas
- propose target layout for runtime-related code
- document what should move now vs later

---

## Includes
- current structure notes
- target structure proposal
- move-now / move-later decision list

---

## Non-Goals
- large file moves in this card
- behavior changes
- API redesign

---

## Acceptance Criteria
- a structure audit doc exists
- target layout is documented
- immediate cleanup candidates are identified
- deferred cleanup candidates are identified
