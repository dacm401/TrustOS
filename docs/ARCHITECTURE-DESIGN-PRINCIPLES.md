# SmartRouter Pro — 信息分发架构设计原则

> "领导不是因为聪明，是因为信息多。"
> 重构了整个问题的前提。

---

## 核心洞察

**行业主流：** 造更强的 AI → 更大模型、更多 context、更强推理

**SmartRouter Pro：** 造更精准的**信息分发层** → 模型能力不再是瓶颈，瓶颈变成"谁决定给谁看什么"

大多数项目在解决"怎么让 AI 更强"。
SmartRouter Pro 在解决"怎么让 AI 的权限和可见信息匹配"。

**这不是能力分层，是信任架构。**

---

## 架构哲学

### 少规则 + 强学习 + 必要时请示用户

```
┌─────────────────────────────────────────────┐
│  Hard Policy（极简红线）                     │
│  - 密码/账号/密钥/私人数据                    │
│  → 直接拒绝，不需要模型判断                  │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│  Learning Layer（学习层）                    │
│  - 从用户反馈中学习分发策略                  │
│  - 记住了就放行/记住了就拦截                 │
│  - 无硬规则覆盖                             │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│  Human-in-the-Loop（用户兜底）              │
│  - 模糊地带 → 请示用户                       │
│  - 用户决定 → 记入学习层                     │
└─────────────────────────────────────────────┘
```

## Phase 0-5 重新解读

| Phase | 实际在做什么 |
|-------|------------|
| Phase 1 | 信息收集层（Worker 拉取上下文） |
| Phase 2 | 信息分类层（L1/L2/L3 分级） |
| Phase 3 | 信息分发层（谁该知道什么） |
| Phase 4 | 信息脱敏层（权限矩阵） |
| Phase 5 | 信息归档层（审计追踪） |

整个 Phase 0-5 不是在搭 Worker 委托流程，**是在搭信息分发拓扑**。

---

## 实现差距

| | 当前实现 | 目标设计 |
|--|---------|---------|
| 决策依据 | LLM 自己判断（无上下文） | LLM + 历史知识库 |
| 规则 | Prompt 里写死（脆弱） | Hard Policy 极少 |
| 反馈闭环 | 无 | 用户决策 → 记忆 → 下次复用 |

---

## 下一步方向

1. **Hard Policy**：提炼极简红线（密码/账号/密钥），代码层面硬编码
2. **Learning Layer**：记录用户每次"批准/拒绝"，构建分发知识库
3. **Human-in-the-Loop**：模糊地带触发用户确认流程
4. **Manager Prompt 改造**：不写死规则，只给"判断框架"
