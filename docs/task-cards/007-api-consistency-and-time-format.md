# Task Card 007: API Consistency and Time Format

## Goal
Improve API consistency with focus on time field format and response predictability.

---

## Scope
- review time fields across task/detail/summary/trace APIs
- standardize format where practical
- document final convention
- avoid unnecessary response churn

---

## Non-Goals
- broad DTO redesign
- frontend-specific formatting
- schema redesign unless required

---

## Acceptance Criteria
- time field convention is defined
- APIs follow the convention or are explicitly documented
- no endpoint regression is introduced
